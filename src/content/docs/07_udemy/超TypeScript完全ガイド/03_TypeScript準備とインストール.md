# TypeScript入門講座：準備編 要約

## 1. フォルダの準備

- 新しく作業用フォルダを作成（例：`TypeScriptレッスン`）。
- このフォルダ内でTypeScriptのコードを作成していく。

---

## 2. エディターの準備

### エディターとは？

- コードを書くためのツール（＝プログラミング用の「ノート」）。

### 推奨エディター

- **Visual Studio Code（VSCode）**
  - 無料で利用可能。
  - TypeScriptのサポートが標準搭載（ビルトイン）。
  - Microsoft製で、TypeScript自体もMicrosoftが開発 → 高い親和性。
  - 実際にVSCodeもTypeScriptで開発されている。

### セットアップ手順

1. VSCodeをダウンロードしてインストール。
2. VSCodeを開き、「ファイル → フォルダーを開く」で先ほど作成したフォルダを指定。

---

## 3. ターミナルの準備

- **ターミナル**：文字ベースでコンピュータを操作するためのツール。
- VSCodeの「表示 → ターミナル」またはショートカットキー（Windowsの場合`Ctrl + J`）で開く。

---

## 4. Node.jsのインストール

### 確認方法

- ターミナルで `npm` コマンドを入力して動作確認。
- もし「npmが見つからない」と表示された場合はNode.jsが未インストール。

### npmとは？

- **Node Package Manager（ノード・パッケージ・マネージャー）**の略。
- Node.jsをインストールすると、自動で一緒にインストールされるツール。
- Node.jsで使う「外部ライブラリ」や「ツール」を管理・インストールするためのコマンド。

### Node.jsとは？

- JavaScriptをPC上で実行できるようにするツール。
- TypeScriptの動作にも必要。

### インストール手順

1. Node.js公式サイトにアクセス。
2. 「Get Node.js」から案内されるコマンドをコピーして実行。

---

## 5. TypeScriptのインストール

### コマンド（グローバルインストール）

```bash
npm install -g typescript
```

- npm: Node.jsのパッケージ管理ツール。
- install: パッケージをインストールする。
- -g: 「グローバル」オプション（どのフォルダでも利用可能にする）。
- typescript: インストール対象のパッケージ名。

### コマンド（ローカルインストール）

```bash
npm init -y
npm install typescript --save-dev
```

### 補足

- 「パッケージ」とはソフトウェアの部品（ツールやライブラリの集まり）のこと。
