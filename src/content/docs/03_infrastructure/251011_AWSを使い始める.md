# AWS のアカウント作成

1. 「AWS」と検索して、スポンサー以外で最上位に出てくる AWS のトップページにアクセスする。

   !["imgs/AWSアカウント作成.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成.png)

2. 画面右上の「アカウントを作成」をクリックする。

   !["imgs/AWSアカウント作成2.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成2.png)

3. メールアドレスとアカウント名を入力して、アカウント作成手続きを進めていく。

   !["imgs/AWSアカウント作成3.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成3.png)

   - 認証コードを用いたメールアドレスの確認
   - ルートユーザーパスワードの設定

   を行う。

4. 無料プランか有料プランか選択する。

   !["imgs/AWSアカウント作成4.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成4.png)

   - 今回は実際に公開するアプリのデプロイを予定しているため、「有料」を選択した。

5. 住所やクレジットカード情報を入力する。

6. サポートプランを選択する。

   !["imgs/AWSアカウント作成5.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成5.png)

   - 今回は自分で試行錯誤することを経験する & 締め切りに追われていないため、無料のベーシックサポートを選択した。

7. アカウントが作成される。作成されるとメールが届く。

   !["imgs/AWSアカウント作成6.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成6.png)

# 初回ログイン

（初回ログインをせずとも、アカウント作成後自動でアカウントでコンソール画面に遷移する。

しかし、アカウント作成で力尽きて PC を閉じた場合、以下のステップを踏んで初回ログインをする）

1. まだ IAM ユーザーは作られていないので、**root ユーザーでログイン**する。

   メールに届いたリンクをクリックして、**ログイン画面を開く**。

   「Sign in using root user email」を押して、アカウント作成時に設定したメールアドレスやパスワードを入力する。

   !["imgs/AWSアカウント作成7.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成7.png)

   !["imgs/AWSアカウント作成8.png"](/_articles/images/imgs_251011_AWSアカウントを使い始める/AWSアカウント作成8.png)

2. root ユーザーの MFA（多要素認証）の設定をする。

   ※初回ログイン後、自動で MFA の設定画面に遷移する場合がある。

   自分のスマホに Google Authenticator をインストールする。

   デバイス名（自分のスマホの名前・Google Authenticator に表示される）を適当に設定する（MFA の設定画面上で）。

   MFA の設定画面上に表示された QR コードを Google Authenticator で読み取り、紐づけをする。

# IAM ユーザの作成

**そもそもなぜ IAM ユーザーを作るの？**

→ root ユーザを普段使いしてはいけないから。

**なぜ root のまま使うのが危険なのか？**

| 理由                            | 説明                                                                              |
| ------------------------------- | --------------------------------------------------------------------------------- |
| ✅ **root は“神権限”**          | すべての AWS サービスに完全アクセス可能（削除も請求も制限なし）。誤操作＝即破壊。 |
| 🧾 **課金・請求・削除もできる** | ちょっとのミスで本番環境ごと消せる。                                              |
| 👥 **アクセスキーも危険**       | もし GitHub に root キーを誤 Push すると、アカウントを乗っ取られる。              |
| 🔒 **AWS の推奨事項**           | AWS 公式が「root は初期設定と請求確認以外使うな」と明記。                         |

→ ✅ Root アカウントは「アカウント開設」「課金・サポートプラン設定」「IAM ユーザー作成」まででログアウト、
以後は IAM ユーザーで運用が基本。
